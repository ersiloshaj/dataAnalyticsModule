<h1>Applied Coding for Data Analytics & Dashboard Development</h1>

<p>
This repository contains work produced as part of a project-based coding module designed to simulate
a real-world data analytics and dashboard development environment.
</p>

<p>
Students will work with realistic, imperfect datasets and are expected to progress from raw data to
cleaned analysis, interactive dashboards, and clear communication of insights.
</p>

<h2>Project Aims</h2>

<ul>
  <li>Develop core coding skills in <strong>R</strong> and <strong>Python</strong></li>
  <li>Build confidence in working with messy, real-world data</li>
  <li>Strengthen analytical and problem-solving thinking</li>
  <li>Introduce reproducible and transparent coding practices</li>
  <li>Simulate an industry or research-style workflow</li>
  <li>Create applied outputs suitable for a professional portfolio</li>
</ul>

<h2>What This Project Simulates</h2>

<p>
This project is framed as a simulated placement or consultancy-style experience.
Students will act as analysts or developers working on commission to deliver a data product.
</p>

<ul>
  <li>Ambiguous and imperfect datasets</li>
  <li>Open-ended analytical questions</li>
  <li>Iterative development and refinement</li>
  <li>Design decisions driven by user needs</li>
  <li>Deadlines, demos, and peer review</li>
</ul>

<h2>End Goal</h2>

<p>
By the end of the project, each student will have produced:
</p>

<ul>
  <li>A functional data analytics dashboard (R Shiny or Python Streamlit)</li>
  <li>Clean, reproducible, and well-documented code</li>
  <li>A structured GitHub repository</li>
  <li>A short technical report describing design decisions and limitations</li>
  <li>A live demo or presentation of their work</li>
</ul>

<h2>Project Structure</h2>

<h3>Weeks 1-2: Data Literacy and R Foundations</h3>

<ul>
  <li>Understanding data structure and tidy data principles</li>
  <li>Cleaning and transforming messy datasets</li>
  <li>Creating clear and interpretable visualisations</li>
  <li>Producing reproducible reports using R Markdown</li>
  <li>Introducing interactivity with R Shiny</li>
</ul>

<h3>Week 3: Python Foundations</h3>

<ul>
  <li>Python syntax and computational thinking</li>
  <li>Writing scripts for data analysis</li>
  <li>Using pandas, numpy, and matplotlib</li>
  <li>Building small analytical utilities</li>
</ul>

<h3>Weeks 3.5-4.5: Version Control and Project Design</h3>

<ul>
  <li>Git and GitHub fundamentals</li>
  <li>Repository structure and commit practices</li>
  <li>Defining project scope and user needs</li>
  <li>Designing dashboards and analytical workflows</li>
</ul>

<h3>Weeks 4-6: Build and Deliver</h3>

<ul>
  <li>Developing backend analytics logic</li>
  <li>Connecting computation to interactive dashboards</li>
  <li>Refining usability and clarity</li>
  <li>Peer review and iterative improvement</li>
  <li>Final demo and presentation</li>
</ul>

<h2>Expected Repository Structure</h2>

<ul>
  <li><code>data/</code> - raw and cleaned datasets</li>
  <li><code>scripts/</code> - data cleaning and analysis scripts</li>
  <li><code>app/</code> - dashboard application files</li>
  <li><code>reports/</code> - technical report or R Markdown outputs</li>
  <li><code>README</code> - project overview and instructions</li>
</ul>

<h2>How This Project Is Assessed</h2>

<ul>
  <li>Clarity and correctness of data processing</li>
  <li>Quality and interpretability of visualisations</li>
  <li>Functionality and usability of the dashboard</li>
  <li>Reproducibility and code organisation</li>
  <li>Ability to explain design decisions and limitations</li>
</ul>

<h2>Philosophy</h2>

<div class="note">
<p>
This project values reasoning and decision-making over perfect answers.
There is rarely a single correct solution.
Students are encouraged to justify their choices, reflect on trade-offs,
and communicate uncertainty clearly.
</p>
</div>

<p>
The work here represents a work in progress for analytical coding,
it is not yet a polished product.
</p>
